# Reading Strategies for Efficient Learning

Evidence-based techniques to maximize learning while minimizing time investment.

---

## Core Principles

### 1. Reading is Not Learning

Reading ≠ Understanding ≠ Remembering ≠ Applying

**The Learning Pyramid:**
```
                    ┌───────────┐
                    │  Reading  │ 10% retention
                    │   only    │
                    ├───────────┤
                    │ Audio/    │ 20% retention
                    │ Visual    │
                    ├───────────┤
                    │Demonstra- │ 30% retention
                    │   tion    │
                    ├───────────┤
                    │Discussion │ 50% retention
                    │  Group    │
                    ├───────────┤
                    │ Practice  │ 75% retention
                    │  Doing    │
                    ├───────────┤
                    │ Teaching  │ 90% retention
                    │  Others   │
                    └───────────┘
```

**Implication:** Passive reading is the least effective. Digests that force you to summarize, question, and apply are more valuable than just highlighting.

### 2. Diminishing Returns

Most value comes from first 20% of effort:

| Time Investment | Knowledge Gained |
|-----------------|------------------|
| First 20% | ~60% of core ideas |
| Next 30% | ~25% more detail |
| Final 50% | ~15% edge cases |

**Implication:** Strategic skimming often beats thorough reading. Know when to stop.

### 3. Forgetting Curve

Without reinforcement, you forget:
- 50% within 1 hour
- 70% within 24 hours
- 90% within 1 week

**Implication:** Create outputs (digests, notes) that enable spaced review.

## The PARA Method for Document Triage

Adapted from Tiago Forte's organizational system:

| Category | Action | Time Investment |
|----------|--------|-----------------|
| **Projects** | Active work - read now, deep | High |
| **Areas** | Ongoing responsibility - selective read | Medium |
| **Resources** | Future reference - skim, save | Low |
| **Archive** | Done/irrelevant - skip | None |

**Before reading, ask:** Which category is this for?

## SQ3R Method (Adapted for Efficiency)

### Survey (2-3 min)
- Title, headings, abstract/introduction
- Figures, tables, diagrams
- Conclusion/summary
- **Output:** Is this worth reading? What's the main claim?

### Question (1 min)
Before deep reading, write 2-3 questions:
- What do I want to learn?
- What problem does this solve for me?
- What's unclear from the survey?

### Read (varies)
- Read with questions in mind
- Stop when questions are answered
- Mark (don't highlight everything):
  - Key claims
  - Evidence for claims
  - Things you disagree with
  - Things you don't understand

### Recite (5 min)
- Close the document
- Write summary from memory
- This is where learning happens
- **The digest creation process IS the recite step**

### Review (later)
- Return to digest periodically
- Update with new understanding
- Connect to other readings

## Strategic Skipping

### What to Skip (Usually)

| Section | Skip When | Read When |
|---------|-----------|-----------|
| **Related Work** | You know the field | You're new to the area |
| **Detailed Methods** | You won't replicate | You need to implement |
| **Proofs/Math** | You trust the result | You need to verify |
| **Appendices** | Core is sufficient | You need raw data |
| **Background** | You have it | You're learning basics |

### Reading Order by Document Type

**Academic Paper:**
```
Abstract → Conclusion → Introduction → Figures → Results → [Methods if needed]
```

**Blog Post:**
```
Title → First paragraph → Headings scan → Full read if passing triage
```

**Technical Article:**
```
Problem statement → Solution overview → Code examples → Details if implementing
```

**News:**
```
Headline → First 2 paragraphs → Skip to quotes → Verify claims
```

## Active Reading Techniques

### Marginalia Method

Don't just highlight - write reactions:
- **?** - I don't understand this
- **!** - Important/surprising
- **→** - Implies/leads to
- **✗** - I disagree
- **∴** - Therefore/conclusion
- **cf.** - Compare to [other work]

### The Feynman Technique

1. Identify concept you don't understand
2. Explain it in simple terms (as if teaching)
3. Find gaps in your explanation
4. Go back to source, fill gaps
5. Simplify and use analogies

**Application:** Your digest summary should be explainable to someone unfamiliar with the topic.

### Question-Based Reading

Turn statements into questions:

| Statement | Question |
|-----------|----------|
| "Transformers outperform RNNs" | Under what conditions? By how much? |
| "We achieved SOTA results" | On which benchmarks? With what baseline? |
| "This approach is more efficient" | In time? Memory? By what factor? |

## Time Boxing Strategies

### Pomodoro for Reading

- 25 min focused reading
- 5 min break + quick summary
- After 4 pomodoros: longer break
- **Total:** ~2 hours max per session

### Time-Limited Triage

| Document Length | Triage Time |
|-----------------|-------------|
| < 5 pages / 2k words | 3 min |
| 5-15 pages / 5k words | 5 min |
| 15-30 pages / 10k words | 10 min |
| > 30 pages | 15 min |

**Rule:** If you can't decide in triage time, default to "not now."

### The "Two Minute Rule"

If a document can be fully processed in < 2 minutes (quick skim, simple answer), do it immediately rather than saving for later.

## Retention Strategies

### Spaced Repetition

Review digests at expanding intervals:
- Day 1: Create digest
- Day 2: Quick review
- Day 7: Review + connect to other knowledge
- Day 30: Review + apply if possible
- Day 90: Review + teach someone

### Interleaving

Don't read all papers on one topic sequentially. Mix:
- Paper on topic A
- Blog on topic B
- Paper on topic A
- Article on topic C

This forces comparison and deeper processing.

### Elaborative Interrogation

For each key claim, ask:
- **Why** is this true?
- **How** does this work?
- **What if** this assumption is wrong?
- **So what** - why does this matter?

## Building a Reading System

### Reading Queue Management

| Priority | Criteria | Action |
|----------|----------|--------|
| **P0** | Directly impacts current work | Read this week |
| **P1** | Important for upcoming work | Read this month |
| **P2** | Generally valuable | Read if time allows |
| **P3** | Interesting but not urgent | Save, review queue monthly |

### Weekly Reading Habit

Suggested allocation for a busy professional:
- **2-3 hours/week** total reading time
- **1 deep read** (major paper/report)
- **3-5 selective reads** (blogs, articles)
- **10-15 skims** (triage, queue management)

### Monthly Review

At month end:
1. Review all digests created
2. Identify connections between readings
3. Create synthesis note if themes emerge
4. Prune reading queue (delete P3s older than 3 months)
5. Adjust priorities based on current focus

## Anti-Patterns to Avoid

### Collector's Fallacy
**Problem:** Saving articles = feeling productive, but no learning happens.
**Solution:** Process or delete. Never save more than you'll realistically read.

### Completionist Trap
**Problem:** Must finish everything you start.
**Solution:** Abandon freely. Your time is valuable.

### Highlighting Illusion
**Problem:** Highlighting feels like learning.
**Solution:** If you can't summarize without looking, you didn't learn.

### Recency Bias
**Problem:** Newest content seems most important.
**Solution:** Include foundational/classic works in reading mix.

### Expert Tunnel Vision
**Problem:** Only reading within your expertise.
**Solution:** Allocate 20% to adjacent fields.

## Integration with doc-digest Skill

The digest creation process embeds these strategies:

| Strategy | How Digest Implements It |
|----------|-------------------------|
| SQ3R Recite | Writing TL;DR and summaries |
| Elaborative Interrogation | "My Assessment" for each claim |
| Active Reading | Critical Evaluation section |
| Spaced Repetition | Action items with future dates |
| Interleaving | Reading multiple docs, creating synthesis |

**The act of creating a digest IS the learning activity.** The output is secondary to the process.
