# Clarity Checklist

Use this checklist after repeated failed debugging attempts:

1. Restate the exact failure signal without interpretation.
2. List the last confirmed working state.
3. List three concrete hypotheses.
4. Note the evidence that would falsify each hypothesis.
5. Identify the fastest verification command for each hypothesis.
6. Record the result of each verification step.
7. Summarize the root cause and the proof that the fix worked.
