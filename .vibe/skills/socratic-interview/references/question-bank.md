# Question Bank — Socratic Interview for Requirements Elicitation

Curated question patterns organized by the five Socratic stages. These are starting points, not scripts. Adapt phrasing to the specific task and conversation flow.

---

## Stage 1 — Definition (What & Why)

Establish core intent. Use when the outcome is vague, the motivation is unclear, or the scope is unbounded.

| Pattern | Example |
|---------|---------|
| Concrete outcome | "When this is done, what will I be able to see, run, or verify?" |
| Problem framing | "What problem does this solve? What happens if we don't do it?" |
| Audience/consumer | "Who will use or read this? What do they care about?" |
| Current state | "What exists today? What's the starting point?" |
| Done definition | "How will you know this is finished and correct?" |

---

## Stage 2 — Elenchus (Test Assumptions)

Surface hidden constraints and contradictions. Use when requirements seem incomplete or internally inconsistent.

| Pattern | Example |
|---------|---------|
| Unstated constraints | "Are there performance, security, or compatibility requirements you haven't mentioned?" |
| Consistency check | "You mentioned X and Y — those seem to pull in different directions. Which takes priority?" |
| Context probe | "What should I know about the existing codebase or conventions that might affect this?" |
| Assumption surfacing | "I'm assuming Z — is that accurate, or should I approach it differently?" |
| Precedent test | "Is there an existing pattern in this project I should follow, or is this a fresh approach?" |

---

## Stage 3 — Maieutic (Elicit Latent Knowledge)

Draw out expertise the user possesses but hasn't articulated. Use when the user has domain knowledge or past experience relevant to the task.

| Pattern | Example |
|---------|---------|
| Past attempts | "Have you tried this before? What worked or didn't?" |
| Reference examples | "Is there an existing implementation you consider a good model for this?" |
| Mental model | "Walk me through how you're thinking about this — what's your mental model?" |
| Preference elicitation | "Do you have a preference for approach, or should I choose?" |
| Knowledge boundary | "How familiar are you with X? That helps me calibrate my explanation depth." |

---

## Stage 4 — Dialectic (Resolve Tensions)

Address competing priorities and tradeoffs. Use when the task involves genuine tensions between desirable properties.

| Pattern | Example |
|---------|---------|
| Priority stack | "If speed, quality, and completeness conflict, what's your priority order?" |
| Scope negotiation | "The ideal solution touches X, Y, and Z. Should I handle all three, or focus on one?" |
| Tradeoff framing | "We can do A (simpler, faster) or B (more robust, more work). Which fits your needs?" |
| Boundary setting | "What is explicitly out of scope? What should I NOT touch?" |
| Incremental vs. complete | "Do you want the full solution now, or a working first pass that we iterate on?" |

---

## Stage 5 — Counterfactual (Validate via Negation)

Confirm understanding by exploring what the result should NOT be. Use for high-stakes tasks or when positive descriptions aren't converging.

| Pattern | Example |
|---------|---------|
| Failure definition | "What would a *wrong* result look like?" |
| Risk identification | "What's your biggest concern about this task?" |
| Minimum viable | "If you had to cut one thing, what's least important?" |
| Anti-pattern check | "Is there a common approach to this that you specifically DON'T want?" |
| Reversal test | "If the result turned out to be X instead of Y, would that still be acceptable?" |

---

## Meta-Questions (Use Sparingly)

These questions operate on the interview itself. Use when the conversation is stuck or drifting.

| Pattern | Example |
|---------|---------|
| Sufficiency check | "I think I have enough to start. Is there anything else I should know?" |
| Reframe | "Let me rephrase — what I'm really trying to understand is..." |
| Priority redirect | "We've covered a lot of ground. What's the single most important thing to get right?" |
| Assumption audit | "Before I summarize, let me tell you what I'm assuming — correct me where I'm wrong." |
