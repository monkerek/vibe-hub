### Current Fetch Tier Architecture

```
Tier 0  Platform CLI         bird (Twitter) · redbook (Red Note)   ← optional, needs .env
Tier 1  Jina Reader          r.jina.ai/<url>                        ← always attempted
Tier 2  twitter-thread.com   Twitter/X only                         ← always attempted
Tier 3  defuddle.md          generic fallback
Tier 4  markdown.new         last-resort direct fetch
Tier 5  Search Synthesis     WebSearch — third-party commentary
```

### Key Design Decisions Made

1. **`.env` over symlinks**: Previous iteration used `~/.config/bird/credentials` symlinked into the repo. Replaced with `.vibe/skills/post-research/.env` (gitignored). Cleaner, more standard, single file for both platforms.

2. **Exit codes for graceful degradation** (all 4 check/read scripts follow this):
   - `0` = authenticated and working (proceed with Tier 0)
   - `2` = not configured (skip silently to Tier 1, NOT an error)
   - `3` = configured but auth failed / tokens expired (skip to Tier 1, warn)
   - `1` = script-level error

3. **Separate vars, combined in scripts**: Twitter uses `TWITTER_AUTH_TOKEN` + `TWITTER_CT0` (not a combined cookie string). Red Note uses `XHS_A1` + `XHS_WEB_SESSION`; scripts combine them into `a1=VAL; web_session=VAL` internally. `XHS_COOKIE_STRING` is still accepted for backwards compatibility.

4. **Tier 0 is completely optional**: The SKILL.md workflow says skip on non-zero exit — the skill functions fully without any credentials, just with degraded fetch quality.
